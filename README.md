# sv

Everything you need to build a Svelte project, powered by [`sv`](https://github.com/sveltejs/cli).

## Creating a project

If you're seeing this, you've probably already done this step. Congrats!

```sh
# create a new project
npx sv create my-app
```

To recreate this project with the same configuration:

```sh
# recreate this project
npx sv@0.16.1 create --template minimal --types jsdoc --add tailwindcss="plugins:typography,forms" --install npm ./
```

## Developing

Once you've created a project and installed dependencies with `npm install` (or `pnpm install` or `yarn`), start a development server:

```sh
npm run dev

# or start the server and open the app in a new browser tab
npm run dev -- --open
```

## Building

To create a production version of your app:

```sh
npm run build
```

You can preview the production build with `npm run preview`.

> To deploy your app, you may need to install an [adapter](https://svelte.dev/docs/kit/adapters) for your target environment.

## Text-to-speech (ElevenLabs)

The flashcards and gallery can read a plant's Latin name aloud (the 🔊 button).
Audio is generated by [ElevenLabs](https://elevenlabs.io), whose neural
multilingual voices pronounce botanical Latin far more naturally than the
browser's built-in speech engine.

**This is optional.** If you don't set an API key, the app automatically falls
back to the browser's built-in voice — nothing breaks, the audio is just lower
quality.

### Setup

1. **Create an ElevenLabs account** at <https://elevenlabs.io> (the free tier
   includes a monthly character allowance, enough for plant names).
2. **Get your API key:** click your profile (bottom-left) → **API Keys** →
   **Create API Key**, and copy it.
3. **Add it to your environment.** Copy `.env.example` to `.env` if you haven't,
   then set:
   ```sh
   ELEVENLABS_API_KEY=sk_your_key_here
   ```
4. **(Optional) Pick a voice.** Open the **Voices** page in ElevenLabs, choose a
   voice, copy its **Voice ID**, and set `ELEVENLABS_VOICE_ID`. If you skip this,
   the default "Rachel" voice is used.
5. **Restart the server** so it picks up the new variables.

### How it works

- The API key lives **only on the server**. The browser never sees it.
- When you press 🔊, the frontend (`src/lib/speech.js`) POSTs the Latin name to
  `/api/speech` (`src/routes/api/speech/+server.js`).
- That endpoint calls ElevenLabs with your key and streams back MP3 audio, which
  the browser plays. Repeated names are cached in memory to save quota.
- If the endpoint isn't configured (no key) or errors, `speech.js` silently
  falls back to the browser voice.

| Variable | Required | Default | Purpose |
|---|---|---|---|
| `ELEVENLABS_API_KEY` | no* | — | Enables ElevenLabs audio. Without it, browser voice is used. |
| `ELEVENLABS_VOICE_ID` | no | `21m00Tcm4TlvDq8ikWAM` (Rachel) | Which voice to speak with. |
| `ELEVENLABS_MODEL_ID` | no | `eleven_multilingual_v2` | Model. Multilingual is needed for Latin. |

\* Required only if you want ElevenLabs-quality audio.
